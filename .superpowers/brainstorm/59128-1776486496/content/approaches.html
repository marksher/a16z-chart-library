Pick a stack for the webapp

All three use Google OAuth + SQLite clickstream. Difference is how the UI is built.

A

Flask + Jinja2 + vanilla JS

Classic server-rendered HTML pages. Form submits → server generates PNG → browser downloads it. Minimal JavaScript — only for the data-entry grid and live preview.

Pros

  • Simplest possible stack
  • Easy to deploy anywhere
  • Everyone knows Flask

Cons

  • Full page reload on submit
  • Less polished UX
B ⭐ recommended

FastAPI + Jinja2 + HTMX

Server-rendered with partial updates. Chart preview refreshes in-place without a full reload. FastAPI handles the PNG/SVG file responses cleanly and has built-in API docs.

Pros

  • Live preview without JS framework
  • Clean binary file handling (PNG/SVG)
  • Type-safe, fast, modern
  • Built-in /docs endpoint

Cons

  • HTMX is a small new thing to learn
  • Slightly more setup than Flask
C

FastAPI + React SPA

Full single-page app with a separate React frontend. API serves JSON + file downloads. Best UX ceiling, but requires a JS build step and two separate codebases to maintain.

Pros

  • Best interactive UX possible
  • Easy to extend later

Cons

  • Overkill for a prototype
  • Node build step required
  • Two codebases to manage
